# Linked Lists

## Description

Problems involving singly or doubly linked list traversal, reversal, merging, and pointer manipulation. The key skill is managing node references without losing track of the list.

## When to Apply

- The input is a linked list or the problem asks you to build one
- You need to reverse, merge, partition, or detect cycles in a sequence of nodes
- In-place modification is required with O(1) extra space
- The problem involves reordering nodes (not values)

## Complexity Characteristics

| Operation | Time | Space |
|---|---|---|
| Traversal | O(n) | O(1) |
| Reversal (iterative) | O(n) | O(1) |
| Fast/slow pointer cycle detection | O(n) | O(1) |
| Merge two sorted lists | O(n + m) | O(1) |
